Live tail requirements
Live tail requires Edge/Chrome with the File System Access API. The app polls the picked log file handle and reads new bytes as it grows.
Picker blocked by “system files”
Browsers may block protected/system locations (commonly parts of %APPDATA% on Windows). Workaround: expose the log under a non-system folder (e.g. Documents/home) using a junction/symlink, then pick the exposed path.
Windows (folder junction example)
if not exist "%USERPROFILE%\Documents\vs-queue-monitor" mkdir "%USERPROFILE%\Documents\vs-queue-monitor"
mklink /J "%USERPROFILE%\Documents\vs-queue-monitor\data-1a2b3c4d" "%APPDATA%\VintagestoryData"
Do not mkdir the data-… / logs-… name first — mklink /J creates it. Then pick: %USERPROFILE%\Documents\vs-queue-monitor\data-1a2b3c4d\Logs\client-main.log — the data-xxxxxxxx suffix is a hash of your pasted path (filled in by “Generate an exact command”).
Linux / macOS (file symlink example)
mkdir -p ~/vs-queue-monitor/logs-9f8e7d6c
ln -s ~/.config/VintagestoryData/Logs/client-main.log ~/vs-queue-monitor/logs-9f8e7d6c/client-main.log
Then pick: ~/vs-queue-monitor/logs-9f8e7d6c/client-main.log — same hash idea from the generator.
Why we can’t show the exact blocked path
For security, browsers don’t reveal the exact filesystem path you attempted to pick when access is blocked. The commands above use typical Vintage Story locations; replace the source path if yours differs.
Generate an exact command (paste your real path)
Paste the Vintage Story data folder path (Windows) or the log file path (Linux), then copy the generated command.
Windows uses a folder junction (mklink /J) into Documents\vs-queue-monitor\ with names like data-xxxxxxxx or logs-xxxxxxxx (8 hex chars from a hash of the source path you paste — same path always gets the same name). Mac/Linux use the same pattern under ~/vs-queue-monitor/.
